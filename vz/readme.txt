Visual Z

技术验证项目
    目的是验证实现最方便使用，让图形界面和脚本一样灵活和容易搭配，提高工作效率。

功能：
1，每个元素都有自己的描述，根据描述可以分析这些程序。
    1，描述是通过最基础的类型来实现。
    2，其他类型则是在此基础上建立的。
2，将运行等都加入到元素中。
    1，Execute元素。
    2，Analysis元素。
    2，Main入口。
3，模型中分为数据和动作。
    1，Action/Data
4，提供各种表现方式，能够绑定模型中的元素。
    1，Representation bind Model
5，所有的元素都保存在统一存储环境中，而不是用文件来保存。

例子：
基本类型Element:
    number  （所有的数字）
    string  （字符串）
    list    （相当于容器）
延伸类型
    table   （表格）
    map     （键-值）
执行动作
    action  （动作）
        赋值
        判断
        循环
        导入/导出
关系（以动态的、瞬间的动作，实现静态的、持久的关系）
    映射  （单向的关系）
    绑定  （双向的关系）
    互斥  （一个成立，其他都不成立）
    同在  （一个成立，其他必须也成立）
    同步  （让两个数据之间保持一致）
    
需求（以简单的、实现为目的的关系，建立高层的、宏观的的要求）
    require     （需要什么）
    管理          （可以对于数据进行删除、添加、修改、排序等等）
    记录历史    （允许数据可以不断的回退）
    灾难恢复    （当出现异常情况后，程序可以退回到安全岛）
    比较          （比较两者不同）
