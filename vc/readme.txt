Visual Command

管理和执行命令的工具。

需求：
总的：希望执行命令能够变得很简单。
1，可以设定命令已经对应的参数。
    1，提供命令输入的帮助，比如现成的模版，或者帮助说明。
    2，提供内部的快捷命令，比如文字替换之类的。
2，可以设定命令运行的列表，并且按照顺序进行执行，发现问题后，就停止。
    1，可以暂停和继续执行。
    2，可以获取已经执行过的命令的日志，并且快速的查找和复制。
    3，可以快速的定制其中哪些命令执行和暂时不执行。
    4，每个命令列表可以设定执行的平台。
3，可以有多个命令列表，每个命令列表设定唯一的名字。
    1，可以创建、删除、修改、保存和加载。
    
    
模板：方便输入命令工具，比如提供workshop/bench下面的命令列表，根据不同平台，以及对应的参数说明。
执行环境：命令执行时的环境，比如平台的名字等。
命令：具体的命令，以及对应的参数等。
命令组：一组命令，可以选择：是否执行、是否在后台等。
命令组动作：对应一个命令组的一次执行。
命令动作：对应一个命令的一次执行。
动作状态：一个动作具有的状态情况，包括是否运行，运行结果，以及中间的日志等。
